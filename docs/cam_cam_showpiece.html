Why a user would want CAM_CAM

Most coding assistants start at the prompt. CAM_CAM starts by understanding the repo universe, then turns that understanding into safer actions.

01

It finds what is already valuable

Instead of asking a novice to know which repo matters, it clusters repos by capability and surfaces reusable systems, risky areas, and promising combinations.

02

It lowers merge risk

Before CAM changes a target repo, Repo Rescue Desk shows where medical, security, key-sensitive, dirty-worktree, and no-test risks exist.

03

It gives CAM memory-shaped inputs

The outputs are not just a report. They are GraphRAG, Logseq, Freeplane, and Markmap artifacts that can become planning and retrieval context.

What is novel

CAM_CAM is moving from chat-assisted coding toward a closed loop: mine, map, plan, build, verify, learn, and reuse.

Capability Typical coding assistant CAM_CAM direction
Repo understanding Usually one repo or current editor context. Cross-repo universe scan with clusters, risks, and build opportunities.
Planning memory Prompt history, rules, or manually maintained notes. Machine-readable artifacts for GraphRAG, Logseq, Markmap, and Freeplane.
Safety before action Often edits first, discovers risk later. Risk triage before mutation, with source repos left untouched.
Self-improvement path Mostly static suggestions unless the user curates lessons. Designed for outcome feedback so successful methods can be promoted and failed ones demoted.

Proof-backed, not just described

The current showpiece is deterministic and testable. It has a local harness, focused tests, and CI passed after the commit.

TEST

Showpiece harness

scripts/test_repo_rescue_desk_showpiece.sh builds fixture repos, runs the CLI, and checks every required artifact exists.

OUTPUT

Real artifacts

The run writes inventory JSON, dashboard HTML, ranked opportunities, risk report, graph JSON, GraphRAG context, and mindmap exports.

GUARD

No target mutation

The first showpiece is intentionally read-only so it can make a useful plan without repeating the earlier mistake of changing the wrong repo.
