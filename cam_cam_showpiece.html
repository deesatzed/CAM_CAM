Read-only triage · 0 source mutations · v1 flagship

Before an AI agent touches your repos, triage the fleet.

Repo Rescue Desk scans a folder of real git repos, clusters them by capability, flags risk before ingestion or mutation, ranks what should be built next, and exports machine-readable context for GraphRAG, Logseq, Markmap, and Freeplane. It is deterministic Python. No LLM calls in the first pass. Cheap, reproducible, safe to run on sensitive local trees.

01 / Why

The hard problem isn't "can an LLM write code." It's which code, learned from what, with what guardrails.

Most AI-coding tools assume the workspace is already curated. In real life — inherited fleets, abandoned spikes, multi-repo mergers, agent-tool experiments — the fleet is the problem. Repo Rescue Desk treats triage as the first action and code mutation as a downstream privilege.

A

Repo sprawl → engineering map

Inventory, cluster by capability, detect duplicates by normalized name, emit a graph a human or agent can actually navigate. 238-repo run produced 249 nodes / 244 edges.

B

Agent-safety footgun, blocked first

Read-only triage reports dirty worktrees, missing tests, security-sensitive language, medical/PII indicators, and API-key-sensitive language before any merge, ingest, or rewrite is allowed.

C

Reusable context, not a PDF

Output is structured for follow-on work: JSON inventory, graph nodes/edges, GraphRAG chunks, Logseq pages, Markmap outline, Freeplane map, dashboard HTML, risk report, and ranked opportunities.

Audience

If you are letting AI agents (Cursor, Devin, Claude Code, Aider, Copilot Workspace, Codex CLI) operate across more than one repo, this is the layer that runs before they get write access. If you are not — most coding assistants already cover the single-repo case.

03 / Risk triage

Conservative heuristics. Routing signals, not verdicts.

Flags are not proof of leaked secrets, PHI exposure, or insecure code. They are routing signals that say "scan deeper before ingestion or mutation." Each flag maps to a concrete downstream action.

FlagCountWhat it detectsRecommended next action
llm-key-sensitive 154 API key, token, secret, OpenRouter, env language in code or docs Run TruffleHog filesystem scan before agent reads. Quarantine repo from auto-ingestion.
security-sensitive 124 Security, sandbox, proxy, audit, credential language Manual review required before allow-listing for build automation.
no-tests 117 No tests directory or test file detected Block cam create --execute until a test gate exists. Demote in build queue.
medical-sensitive 72 Medical, clinical, patient, HIPAA, PHI, PII language Hard stop on cloud LLM calls. Route to local-only (Ollama / MLX-LM) pipeline.
dirty-worktree 58 Uncommitted or untracked changes Refuse mutation. Surface diff for human commit/stash before agent operates.
no-readme 7 Purpose not self-evident from README Defer mining. README synthesis pass before clustering.

04 / How

Eight deterministic stages. Zero LLM calls in the first pass.

01

Discover

Walk one root directory, select direct child git repos, skip node_modules, .venv, .next, target, build caches.

02

Profile

Read README text, count language extensions, inspect git remote, branch, short HEAD, last commit date, dirty state, untracked file count.

03

Cluster

Score repo name + README against eight domain keyword families. Assign strongest cluster or mark as uncategorized when no signal is present.

04

Risk-flag

Detect dirty worktree, no README, no tests, medical/PII terms, security terms, API-key terms, large Node-dependency risk.

05

Graph

Create cluster nodes, repo nodes, belongs_to edges, duplicate-name group edges. Real run: 249 nodes, 244 edges.

06

Rank

Score showpiece opportunities by support across required clusters. Attach supporting repo evidence and measurable outcome targets.

07

Export

Write dashboard HTML, JSON inventory, graph JSON, GraphRAG markdown, Markmap outline, Freeplane XML, Logseq pages, risk report, opportunities markdown.

08

Preserve

Write per-repo scan receipts so future CAM steps can prove which source state was inspected before any action was taken.

06 / Build queue

What the scan suggests building next.

The opportunity score is support count — for each candidate build, how many repos in the fleet already cover its required capability clusters. A higher score means more existing material to mine, lower greenfield cost. It is simple, explainable, and auditable; it is not independent product/market validation.

The ranking is "what's cheapest to build given what you already have," not "what should exist in the world." Read it that way.

07 / Vs

How this differs from coding assistants.

Repo Rescue Desk is not trying to beat autocomplete at typing. It is trying to decide what context and safeguards should exist before an AI engineer acts. Some assistants do touch this — Sourcegraph Cody and Devin both have cross-repo capabilities — but none publish a deterministic, read-only triage layer with risk gating as the first action.

CapabilityTypical SWE assistantRepo Rescue Desk
Scope Current repo, open files, manually indexed workspace. Cross-repo universe. Scanned 238 local repos, produced a graph.
First action Often edits code immediately after a prompt. Read-only triage: risks, clusters, duplicate groups, opportunities, receipts.
Memory shape Chat memory, rules files, generated summaries. Agent-ingestible artifacts: graph JSON, GraphRAG chunks, Logseq, Markmap, Freeplane.
Evidence Usually no reproducible harness for landing-page claims. Harness-backed. scripts/test_repo_rescue_desk_showpiece.sh verifies the artifact set.
Safety posture Risk discovered when tests fail or a human notices. Risk flags before mutation: dirty worktrees, no tests, medical/PII, API keys.
Next-step planning User must know what to ask for next. Ranks next builds with supporting repos and measurable outcomes.

09 / Run

Try it on your own repo fleet.

Three steps. Source repos are not modified. Output lives entirely under tmp/repo_rescue_desk/latest.

1

Clone the repo.

git clone https://github.com/deesatzed/CAM_CAM.git
cd CAM_CAM
2

Run the scanner against your folder of git repos.

PYTHONPATH=apps/repo_rescue_desk python -m repo_rescue_desk.cli \
  --root /path/to/folder/containing/git/repos \
  --out-dir tmp/repo_rescue_desk/latest
3

Open the dashboard, inspect the artifacts.

open tmp/repo_rescue_desk/latest/dashboard.html
cat tmp/repo_rescue_desk/latest/risk_report.md
cat tmp/repo_rescue_desk/latest/opportunity_rankings.md

Tested on macOS (Apple Silicon, Python 3.11+). The reference run used /Volumes/WS4TB/repo421sn, a 238-repo heterogeneous fleet. Zero source mutations. Zero LLM calls in the first pass.
